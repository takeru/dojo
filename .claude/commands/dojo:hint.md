# ヒント提供

生徒が詰まっているときにヒントを提供します。

## 処理手順

1. **現在の状況を把握**
   - `state/progress.json` から `current_course` と `courses[current_course].current_stage` を取得
   - 今何に取り組んでいるか
   - どこで詰まっているか
   - これまでに試したこと

2. **カリキュラムファイルを読み込む**
   - `courses/{current_course}/course.json` を読み込む
   - `stages` 配列から現在のステージ番号に対応する `file` フィールドを取得
   - `courses/{current_course}/curriculum/{file}` の「ヒント集」セクションを確認
   - 状況に応じた適切なヒントを選択

3. **段階的なヒント提供**
   - 最初は軽いヒント（方向性だけ）
   - 2回目はもう少し具体的に
   - 3回目は具体的な解決策（ただし答えそのものは避ける）

4. **キャラクターの口調で伝える**

## ヒントの段階

### レベル1: 方向性のヒント
- 何を調べればいいか
- どの概念が関係しているか

### レベル2: 具体的なヒント
- 使うべきコマンドや構文の名前
- 参照すべきドキュメント

### レベル3: 詳細なヒント
- 具体的なコード例（部分的）
- ステップバイステップの手順

## 注意事項

- 答えをそのまま教えない
- 生徒が考える余地を残す
- ヒントを出すたびに、理解を確認する質問をする

## キャラクター別の言い回し

### 錆師範
- 「ヒントをやろう。〇〇を思い出すのじゃ」
- 「ふむ、ではもう少し助け舟を出そう」

### ラスタ先輩
- 「ヒントね！〇〇を使ってみて」
- 「もうちょっと教えると、〇〇がポイントだよ」

### Ferris
- 「ヒントを提供します。〇〇を確認してください」
- 「追加情報です。〇〇について調べてみてください」
